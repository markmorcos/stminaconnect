# Internationalization (i18n)

## Setup

| Aspect        | Choice                                                 |
| ------------- | ------------------------------------------------------ |
| Library       | i18next + react-i18next                                |
| Detection     | Device locale → AsyncStorage persisted choice          |
| Fallback      | English (en)                                           |
| Interpolation | `{{name}}` syntax                                      |
| RTL           | Automatic layout flipping for Arabic via `I18nManager` |
| Font          | Cairo (Google Fonts) — covers Arabic + Latin scripts   |

### Configuration (existing in `src/i18n/index.ts`)

```typescript
import i18n from "i18next";
import { initReactI18next } from "react-i18next";
import * as Localization from "expo-localization";
import AsyncStorage from "@react-native-async-storage/async-storage";

i18n.use(initReactI18next).init({
  lng: (await AsyncStorage.getItem("language")) || Localization.locale.split("-")[0],
  fallbackLng: "en",
  supportedLngs: ["en", "ar", "de"],
  interpolation: { escapeValue: false },
  resources: { en, ar, de },
});
```

## RTL Handling

| Rule             | Details                                                                             |
| ---------------- | ----------------------------------------------------------------------------------- |
| Layout direction | `I18nManager.forceRTL(true)` when `ar` selected. Requires app restart.              |
| Flex direction   | Use `flexDirection: I18nManager.isRTL ? 'row-reverse' : 'row'` OR use `Start`/`End` |
| Padding/margin   | Always use `paddingStart`/`paddingEnd`, never `paddingLeft`/`paddingRight`          |
| Text alignment   | `textAlign: 'auto'` (respects language direction automatically)                     |
| Icons            | Directional icons (arrows, chevrons) flip automatically with RTL                    |
| Navigation       | Back gesture/animation flips automatically with Expo Router                         |
| Cairo font       | Arabic line height multiplier: 1.1x (already configured in theme)                   |

### Language Switch Flow

1. User selects language in Settings
2. Persist to AsyncStorage
3. Update i18n instance: `i18n.changeLanguage(lang)`
4. If switching to/from Arabic: `I18nManager.forceRTL(lang === 'ar')` + restart
5. All components re-render via react-i18next context

## Translation Key Structure

Namespace-based with dot notation. Keys organized by feature area.

```
src/i18n/
  index.ts          # i18n configuration
  en.json           # English translations
  ar.json           # Arabic translations
  de.json           # German translations
```

### Namespace Hierarchy

```
common.*            # Shared actions: save, cancel, delete, edit, etc.
tabs.*              # Tab bar labels
auth.*              # Login & onboarding
home.*              # Dashboard screen
checkin.*           # Check-in flow
people.*            # Registration & member management
followUp.*          # Follow-up flow
settings.*          # Settings screen
sync.*              # Sync status messages
notifications.*     # Notification strings (for in-app display)
reports.*           # Report screen labels
```

## Date/Time/Number Formatting

| Locale | Date          | Time    | Numbers |
| ------ | ------------- | ------- | ------- |
| en     | Mar 31, 2026  | 9:00 AM | 1,234   |
| ar     | ٣١ مارس ٢٠٢٦  | ٩:٠٠ ص  | ١٬٢٣٤   |
| de     | 31. März 2026 | 9:00    | 1.234   |

Use `Intl.DateTimeFormat` and `Intl.NumberFormat` with the active locale. These are built into modern JS engines and Hermes.

```typescript
// Example
const formatDate = (date: Date, locale: string) =>
  new Intl.DateTimeFormat(locale, { dateStyle: "medium" }).format(date);

const formatTime = (date: Date, locale: string) =>
  new Intl.DateTimeFormat(locale, { timeStyle: "short" }).format(date);
```

## Notes

- **Member data is language-agnostic**: Names, comments, regions are stored as-entered by the user, regardless of app language. They are not translated.
- **Quick Add greeting adapts in real-time**: When the newcomer switches the language selector on the Quick Add form, the greeting and labels change immediately without page reload.
- **Notification text is generated server-side** in the recipient's `preferred_language` (from `servants` table).
